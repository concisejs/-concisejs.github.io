项目配置

ConciseJS 项目的配置项放在了 package.json 的 conciseConfig.Application 选项中。

ConciseJS 希望尽可能地减少开发者对框架的学习成本，所以并没有对外暴露大量的配置项，也没有提供一个单独的配置文件，但实际业务中往往有一些需要额外配置或修改的配置项，如果有这个需求，可以在 package.json 的 conciseConfig.Application 选项中进行配置。

package.josn
{
    // ...
    "dependencies": {
        "concisejs": "^0.7.2"
    },
    "conciseConfig": {
        "Application": {
            "port": 8080,
            "baseUrl": "/项目名/",
            "buildDir": "dist/Web/项目名",
            "releaseEnable": true,
            "proxy": {},
            "statics": []
        }
    }
}

下面，对常用到的配置项做简单的介绍。

port​

  • 描述：开发模式下应用程序运行的端口号
  • 类型：Number/String
  • 默认：8080

baseUrl​

  • 描述：打包部署时的基本 URL
  • 类型：String
  • 默认：/项目名/，比如人脸项目 /Face/

buildDir​

  • 描述：打包目录
  • 类型：String
  • 默认：dist/Web/项目名，比如人脸项目 dist/Web/Face

releaseEnable​

  • 描述：是否允许写入 release 文件
  • 类型：Boolean
  • 默认：true

proxy​

  • 描述：设置代理
  • 类型：Object
  • 默认：{}

注意：在生产环境下，代理是无法生效的，所以本项设置只针对开发环境。

在开发模式下，我们常常会用到模拟接口，这时就需要配置代理：

package.josn
{
    // ...
    "dependencies": {
        "concisejs": "^0.7.2"
    },
    "conciseConfig": {
        "Application": {
            "proxy": {
                "/TTS/api": {
                    "target": "http://192.168.1.36:3001/mock/147",
                    "pathRewrite": {
                        "^/TTS/api": "/api"
                    }
                },
                "/api": {
                    "target": "http://192.168.1.36:3001/mock/147" // 请求 Yapi 接口
                }
            }
        }
    }
}

增加 proxy 配置的目的是为了去掉原 public/config.js 文件中的配置。设置 proxy 后，开发模式下可以正确请求模拟接口；项目放到部署环境后， ConciseJS 会将接口请求 /api/xxx 处理为 http://host:port/项目名/api/xxx ，如果 API 与前端应用不在一台服务器上，则配置 nginx 将 /项目名/api 代理到需要的服务器上。

statics​

  • 描述：需要插入的静态资源（CSS、JavaScript）
  • 类型：Array.<String>
  • 默认：[]

通过此项配置，可以将一些共用的静态资源文件插入到 HTML 中（ CSS 插入到 head 中， JavaScript 插入到 body 中）：

package.josn
{
    "conciseConfig": {
        "Application": {
            "statics": [
                "/cdn/leaflet-1.7.1/leaflet.css",
                "/cdn/leaflet-1.7.1/leaflet.js"
            ]
        }
    }
}
