鉴权控制

项目本身支持系统鉴权，默认除 login 、 403 和 404 页面外，其他页面都需要系统鉴权。

默认系统鉴权采用的是我们内部一直约定的方式，但也支持对系统鉴权做定制化处理和灵活配置页面的系统鉴权。

注：系统鉴权与路由权限不同，由于我们的应用场景中，路由权限的控制是灵活配置的，所以 ConciseJS 项目自身不支持路由权限控制，而是通过接口来控制路由权限。

默认系统鉴权​

默认系统鉴权是通过识别 cookie 和 验证 cookie 的方式实现的，分两种情况：

  • 用户未登录时，未识别到 cookie ，则跳转至登录页面，然后进行登录。
  • 用户已登录或单点登录时，验证当前 cookie 信息是否有效，如果有效则进入相应的路由，如果无效则跳转至登陆页面。

注意： cookie 信息只保留会话 ID ，不再保留用户信息、路由信息、权限等。当验证 cookie 信息是否有效时，是验证当前会话 ID 是否有效，通过请求身份验证接口来实现（同时，如果当前会话 ID 有效，接口会返回用户信息、已授权的路由、菜单、配置等）。

登录​

import {useHistory, useDispatch, useSelector} from 'concisejs'

export default function Login() {
    const history = useHistory()
    const dispatch = useDispatch()
    const {_auth} = useSelector(state => state)

    // ...

    const doLogin = async () => {
        const loginInfo = {username: '', password: ''}
        await dispatch({
            type: '_auth/handleLogin',
            payload: loginInfo
        }).then(() => {
            history.push('/')
        }).catch(err => {})
    }

    // ...
}

身份验证​

import {useHistory, useDispatch, useSelector} from 'concisejs'

export default function Login() {
    const history = useHistory()
    const dispatch = useDispatch()
    const {_auth} = useSelector(state => state)

    // ...

    const doAuth = async () => {
        const authToken = _auth.authToken
        await dispatch({
            type: '_auth/handleAuth'
        }).then(() => {
            history.push('/')
        }).catch(err => {})
    }

    // ...
}

注：由于 ConciseJS 是单页面应用，故身份验证只会触发一次。

开启定制化系统鉴权​

当存在 src/pages/_Auth.jsx 文件时，将开启定制，此时，系统鉴权以此文件的逻辑处理为准，默认系统鉴权失效。

src/pages/_Auth.jsx 返回一个 React 组件，并通过 props.children 渲染子组件。

定制化系统鉴权示例​

关闭系统鉴权​

src/pages/_Auth.jsx
export default function Auth(props) {
  return <>{props.children}</> // 直接放行，意味着所有路由都不需要登录验证
}

修改鉴权逻辑​

src/pages/_Auth.jsx
import {useHistory, useLocation, useState} from 'concisejs'

const Loading = (props) => {
  return <div>加载中...</div>
}

export default function Auth(props) {
  const history = useHistory()
  const location = useLocation()
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    if (...) {
      // ...
      setLoading(false)
    } else {
      setLoading(false)
      if (location.pathname !== '/login') {
        history.push('/login')
      }
    }
  }, [])

  return (
    loading
    ?
    <Loading />
    :
    <>{props.children}</>
  )
}

控制特定路由​

src/pages/_Auth.jsx
import {useHistory, useLocation, useState} from 'concisejs'

const Loading = (props) => {
  return <div>加载中...</div>
}

export default function Auth(props) {
  const history = useHistory()
  const location = useLocation()
  const [loading, setLoading] = useState(true)

  const isReleaseRoute = ['/', '/home'].includes(location.pathname)

  useEffect(() => {
    if (!isReleaseRoute) {
      if (...) {
        // ...
        setLoading(false)
      } else {
        setLoading(false)
        if (location.pathname !== '/login') {
          history.push('/login')
        }
      }
    }
  }, [])

  // 对特定路由放行
  if (isReleaseRoute) {
    return <>{props.children}</>
  }

  return (
    loading
    ?
    <Loading />
    :
    <>{props.children}</>
  )
}
