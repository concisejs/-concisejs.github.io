目录结构

项目创建完成后，目录结构是这样的： 

/myApp/
    |-- node_modules/
    |-- src/
    |    |-- assets/
    |    |-- components/
    |    |-- config/ # 默认无此目录
    |    |-- models/
    |    |-- pages/
    |    |-- services/
    |
    |-- .editorconfig
    |-- .gitignore
    |-- .npmrc
    |-- package-lock.json
    |-- package.json
    |-- README.md
    |-- yarn.lock

该目录与常见的单页面应用目录有所差别，这里没有 public 文件夹和入口文件 src/index.jsx 。

通常 public/index.html 是页面的模板文件， ConciseJS 框架内置了页面模板，不需要额外的模板文件，而且 ConciseJS 也不推荐在模板文件 public/index.html 上追加内容，所以去掉了页面模板文件；通常 public/static 是项目中用到的第三方静态资源，这里也不再保留此目录， ConciseJS 推荐各个项目使用一个公共的第三方静态资源目录，而不是每个项目一个第三方静态资源目录。

由于 ConciseJS 的目的是尽可能的降低开发成本，所以把入口文件 src/index.jsx 也去掉了， ConciseJS 会根据 src/pages/ 下的页面及其他路由、状态管理等自动生成入口文件，开发者只需进行页面开发即可。

node_modules 目录​

node_modules/ 目录是执行安装依赖命令（npm i）后，用来存放依赖包的文件夹。

src 目录​

src/ 是开发目录，所有的代码开发都是在这个目录下进行的。

其中：

  • src/assets/ 目录：用来存放开发中的静态资源文件，如项目公共样式、主题配置、字体文件、图片等。
  • src/components/ 目录：组件目录。
  • src/config/ 目录：配置目录，项目默认没有此目录，当需要配置式路由功能时，会用到此目录。
  • src/models/ 目录：JavaScript 状态管理目录。
  • src/pages/ 目录：页面目录。
  • src/services/ 目录：数据请求目录。

根目录​

.editorconfig​

.editorconfig 文件定义项目的编码规范，它可以帮助开发人员在不同的编辑器和 IDE 之间定义和维护一致的编码样式。很多编辑器和 IDE 无需插件直接支持，如果你正在使用 Visual Studio Code 编辑器，请安装插件以支持 EditorConfig 。

.gitignore​

用来忽略 Git 中不想提交的文件。在该文件中定义相应的忽略规则，来管理当前文件夹下的文件的 Git 提交行为。

.npmrc​

设置当前目录下的 npm 配置。

package-lock.json​

npm 的 lock 文件，它准确的描述了当前项目的依赖包树。

package.json​

定义了这个项目所需要的各个依赖，以及项目的配置信息（比如名称、版本、许可证等元数据）。

README.md​

项目的入门手册。

yarn.lock​

yarn 的 lock 文件。
