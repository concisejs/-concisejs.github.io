状态管理

ConciseJS 提供了一种基于 Redux 的 JavaScript 状态管理解决方案，可以极大程度的简化应用程序的状态和逻辑管理。

尽管 ConciseJS 简化了应用共享状态管理，但是并不是所有的应用场景都需要它，通常在以下场景中，可以考虑使用：

  • 某个组件的状态，需要共享
  • 嵌套多层的组件间的传值
  • 界面操作十分复杂

使用方式​

在前面目录结构章节，我们了解到 ConciseJS 项目的状态管理目录在 src/models/ 下，这和以往项目中的 store 目录大不相同，在这个目录下，不再需要拆分 actions.js 、 reducers.js 和 store.js 文件，而是将一组数据和逻辑放到一个文件中。

以下是一个状态管理的示例：

src/models/user.js
import {login} from '@/services/common'

const initialUser = localStorage.getItem('user') ?
  JSON.parse(localStorage.getItem('user')) :
  {}

const user = {
  name: 'user',
  initialState: {
    /**
     * @param {Object} user 用户信息
     * @param {String} user.name 用户名
     * @param {String} user.uid 用户 ID
     * @param {String} user.text 姓名
     * @param {String} user.phone 手机号
     */
    user: initialUser
  },
  reducers: {
    loginSuccess: (state, action) => {
      state.user = action.payload
      localStorage.setItem('user', JSON.stringify(action.payload))
      // return {...state, user: action.payload || {}}
    },
    logoutSuccess: (state, action) => {
      state.user = null
      localStorage.removeItem('user')
      // return {...state, user: {}}
    },
    say: (state, action) => {
      console.log('reducer\'s `say` .')
      // return state
    }
  },
  effects: {
    doLogin: async({getState, dispatch}, {payload}) => {
      // the example usage of `getState`
      const state = getState()
      console.log('state.user.user:', state.user.user)
      // the example usage of `dispatch`
      dispatch({
        type: 'user/say'
      })
      // effect
      // payload = {username, password}
      const mockLoginRes = await login(payload)
      // the example usage of `dispatch`
      dispatch({
        type: 'user/loginSuccess',
        payload: mockLoginRes.data
      })
    }
  }
}

export default user

我们在 src/models 文件夹中定义了 model 以后，可以直接在页面中使用 hooks 来操作：

src/pages/Login.jsx
import {useDispatch, useSelector} from 'concisejs'

export default function Login(props) {
  const dispatch = useDispatch()
  const {user} = useSelector(state => state.user)
  // 这里相当于是
  // const userAllState = useSelector(state => state.user)
  // const user = userAllState.user

  const handleLogin = async () => {
    const username = document.querySelector('#username').value
    const password = document.querySelector('#password').value
    await dispatch({
      type: 'user/doLogin',
      payload: {username, password}
    })
  }

  return (
    <div>
      <input type="text" name="username" id="username" placeholder="请输入用户名" />
      <input type="password" name="password" id="password" placeholder="请输入密码" />
      <button type="button" onClick={handleLogin}>登录</button>
    </div>
  )
}

model 说明​

name​

model 的名称，这是它在全局 state 上的属性名，只能用字符串。一个项目中不能有重复名称的 model ，建议将文件名与 name 值保持一致，这样就能直接的通过文件命名来避免这个问题。

initialState​

当前 state 的初始值。

reducers​

reducers 是 reducer 的集合。 reducer 也叫 reducing function [1] ，通常 reducer 是一个纯函数，纯函数里不能改变 state ，必须返回一个全新的对象。但是 ConcsieJS 在内部使用 Immer 来允许修改 state ，您可以通过简单地修改数据而与数据进行交互，同时保留不可变数据的所有优点。

reducers 是唯一可以修改 state 的地方。

effects​

effects 是 effect 的集合。 effect 被称为副作用，用于处理异步操作和业务逻辑。与 reducer 一样，由 action 触发，但是不可以修改 state ，可以触发 action ，可以获取 state 。

内置 model​

_auth​

ConciseJS 内置身份验证 model（ _auth model ），可以方便地进行用户信息获取、登录、身份验证等操作。

_auth model 包含的内容有：

const _auth = {
  name: '_auth',
  initialState: {
    /**
     * @param {String} authToken 当前的会话 ID ，使用 session_id 来确认客户端身份及登录状态
     */
    authToken: '',
    /**
     * @param {Object} user 用户信息
     * @param {String} user.name 用户名/登录账户
     * @param {String} user.text 姓名
     * @param {String} user.uid 用户 ID
     * @param {String} user.phone 手机号
     * @param {String} user.lastIp 上次登录 IP
     * @param {String} user.lastTime 上次登录时间
     * @param ...
     */
    user: {},
    /**
     * @param {String[]} routes 已授权的路由列表
     */
    routes: [],
    /**
     * @param {Object[]} menus 导航列表
     * @param {String} menus[].name
     * @param {String} menus[].path
     * @param {String} menus[].text
     * @param {Boolean} menus[].external 是否为外部链接，默认 false
     * @param {Boolean} menus[].blank 是否新页面打开，默认 false
     */
    menus: [],
    /**
     * @param {Object} config 配置
     */
    config: {}
  },
  reducers: {
    // 略...
  },
  effects: {
    /**
     * @description 登录操作，请求登录接口 `/api/login`
     */
    handleLogin: async () => {},
    /**
     * @description 身份验证，并返回相关信息（用户信息、已授权的路由、菜单、配置等），请求身份验证接口 `/api/auth`
     */
    handleAuth: async () => {}
  }
}

登录、身份验证接口（ /api/login 、 /api/auth ）返回数据格式相同：

{
  "status": 0, // 0 成功 1 失败
  "data": {
    "user": "", // String 类型。使用 AES 算法加密后的用户信息（使用 AES 算法对用户信息的 JSON 字符串进行加密）
    "routes": [], // Arrary 类型。已授权的路由
    "menus": [], // Array 类型。菜单
    "config": "" // String 类型。使用 AES 算法加密后的配置（使用 AES 算法对配置的 JSON 字符串进行加密）
  }
}

登录、身份验证接口（ /api/login 、 /api/auth ）的具体示例用法见 鉴权控制 - 默认系统鉴权 部分。




名词解释：

[1] reducing function ： reducer 也叫 reducing function ，当状态容器收到 action 以后，必须给出一个新的 state ，这样 view 才会发生变化，这种 state 的计算过程就叫做 reducer 。它是一个函数，它接受 action 和当前 state 作为参数，返回一个新的 state 。
