配置式路由

ConciseJS 项目支持配置式路由。

ConciseJS 项目是单页应用，页面地址的跳转都是在浏览器端完成的，不会重新请求服务器端获取 html ， html 只在页面初次加载时加载一次。所有页面由不同的组件构成，页面的切换其实就是不同组件的切换，只需在路由配置中把不同的路由路径和对应的组件关联上即可。

路由配置文件​

路由配置文件位于 src/config/routes.config.js ，当该文件存在时， ConciseJS 会关闭约定式路由功能，即路由配置文件的优先级比约定式路由高。

一份常见的路由配置如下：

src/config/routes.config.js
export default [
  {
    path: '/',
    component: '@/pages/index.jsx',
    exact: true,
    requireAuth: false
  },
  {
    name: 'one',
    routes: [
      {
        path: '/one',
        component: '@/pages/One/index.jsx',
        exact: true,
        requireAuth: true
      },
      {
        path: '/one/person',
        component: '@/pages/One/Person.jsx',
        exact: true,
        requireAuth: true
      }
    ]
  }
]

路由集合​

name​

  • 描述：该路由集合的名称
  • 类似：String

routes​

  • 描述：配置子路由
  • 类型：Array.<Object>

路由项​

path​

  • 描述：路由路径
  • 类型：String

component​

  • 描述：组件路径，即 src/pages/** ，统一以 @ 的方式书写，如 @/pages/One/Person.jsx
  • 类型：String

exact​

  • 描述：是否严格匹配，即 location 是否需要和 path 完全对应
  • 类型：Bolean
  • 默认：true

比如：

export default [
  {
    path: '/one', // url 为 /one/person 时匹配失效，只能在 url 为 /one 时才匹配成功
    exact: true
  },
  {
    path: '/one', // url 为 /one/person 时匹配成功
    exact: false
  }
]

requireAuth​

  • 描述：是否需要权限验证
  • 类型：Boolean
  • 默认：false
