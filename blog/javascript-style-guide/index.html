    • String

    • null
    • undefined
    • Symbol
  • 引用类型
    • Object

声明#

  • var
  • let

规范#

声明变量#

  • 使用 const 或 let 声明变量，避免使用 var 。

    这样做不会导致全局变量，避免污染全局命名空间。

    // bad
    var a = 1;
    // good
    const a = 1;

  • 所有的声明都使用 const ，如果需要对变量重新赋值，使用 let 。

    这样做能确保你不会改变你的初始值，重复引用会导致 bug ，并使代码变得难以理解。

    // error
    const count = 1;
    if (true) {
    count += 1;
    }
    // good
    let count = 1;
    if (true) {
    count += 1;
    }

  • 声明每一个变量都用一个 const 或 let 。

    这种方式很容易去声明新的变量，不用去考虑把 ; 调换成 , ，这种做法也可以在调试代码时单步调试每个声明语句，而不是一下跳过所有声明。

    // bad
    const items = getItems(), isAdd = true, name = 'admin';
    // good
    const item = getItems();
    const isAdd = true;
    const name = 'admin';

  • 把 const 和 let 分别放在一起。

    这种做法有助于变量的分类管理。

    // bad
    let i;
    const items = getItems();
    let isAdd = true;
    let len = 10;
    const name = 'admin';
    // good
    const items = getItems();
    const name = 'admin';
    let i;
    let len = 10;
    let isAdd = true;

  • 在你需要的地方声明变量，但是要放在合理的位置。

    const 和 let 都是块级作用域，而不是函数级作用域。

    // bad
    function checkName(hasName) {
    const name = getName();
    if (hasName) {
    return false;
    }
    if (name === 'test') {
    return false;
    }
    return true;
    }
    // good
    function checkName(hasName) {
    if (hasName) {
    return false;
    }
    // 在需要的时候分配
    const name = getName();
    if (name === 'test') {
    return false;
    }
    return true;
    }

  • 不要使用链式声明变量。

    链式声明变量会创建隐式全局变量。

    // bad
    function example() {
    let a = b = c = 1;
    }
    example();
    console.log(a); // error: a is not defined
    console.log(b); // 1
    console.log(c); // 1
    // good
    function example() {
    let a = 1;
    let b = 1;
    let c = 1;
    }
    example();
    console.log(a); // error: a is not defined
    console.log(b); // error: a is not defined
    console.log(c); // error: a is not defined
    // `const` 也是如此

  • 不允许存在未使用的变量。

    一旦声明了未使用的变量，很像是有种功能未实现或未完成的感觉，容易给代码阅读者造成迷惑。

    // bad
    let name = 'string';
    let count = 1;
    if (true) {
    count += 1;
    }
    // good
    let count = 1;
    if (true) {
    count += 1;
    }

对象#

  • 使用字面值创建对象。

    这是创建对象最简单的方式。

    // bad
    const obj = new Object();
    // good
    const obj = {};

  • 使用计算属性名创建一个带有动态属性名的对象。

    这可以在一个地方定义所有对象属性。

    function getKey(val) {
    return `a key named ${val}`
    }
    // bad
    const obj = {
    id: 5,
    name: 'string'
    };
    obj[getKey('test')] = true;
    // good
    const obj = {
    id: 5,
    name: 'string',
    [getKey('test')]: true
    };

  • 只对那些无效的标识使用单引号 '' 。

    这种方式更易读，不仅优化了代码高亮，也更容易被许多 JS 引擎优化。

    // bad
    const obj = {
    'one': 1,
    'two': 2,
    'data-blah': 5
    };
    // good
    const obj = {
    one: 1,
    two: 2,
    'data-blah': 5
    };

  • 用对象方法简写。

    这样写的更少且可读性更高。

    // bad
    const atom = {
    value: 1,
    addValue: function(val) {
    return atom.value + val;
    }
    };
    // good
    const atom = {
    value: 1,
    addValue(val) {
    return atom.value + val;
    }
    };

  • 用属性值缩写。

    这样写的更少且可读性更高。

    const username = 'admin';
    const password = '123456';
    // bad
    const obj = {
    username: username,
    password: password
    };
    // good
    const obj = {
    username,
    password
    };

  • 对象浅拷贝时，更推荐使用扩展运算符（即 ... 运算符），而不是 Object.assign 。

    使用 Object.assign 不当会影响原始变量。

    // very bad
    const original = {a: 1, b: 2};
    const copy = Object.assign(original, {c: 3}); // original => {a: 1, b: 2, c: 3}
    delete copy.a; // original => {b: 2, c: 3}
    // bad
    const original = {a: 1, b: 2};
    const copy = Object.assign({}, original, {c: 3});
    // good
    const original = {a: 1, b: 2};
    const copy = {...original, c: 3};

  • 用对象的解构赋值来获取和使用对象某个或多个属性值。

    解构使您不必为这些属性创建临时引用，并且避免重复引用对象。重复引用对象将造成代码重复、增加阅读次数、提高犯错概率等。

    // bad
    function getFullName(user) {
    const firstName = user.firstName;
    const lastName = user.lastName;
    return `${firstName} ${lastName}`;
    }
    // good
    function getFullName(user) {
    const {firstName, lastName} = user;
    return `${firstName} ${lastName}`;
    }
    // best
    function getFullName({firstName, lastName}) {
    return `${firstName} ${lastName}`;
    }
    // rest 解构运算符
    const original = {a: 1, b: 2, c: 3};
    const {a, ...copy} = original; // copy => {b: 2, c: 3}

数组#

  • 使用字面量创建数组。

    这是创建数组最简单的方式。

    // bad
    const items = new Array();
    // good
    const items = [];

  • 使用 Array.push 代替直接向数组中添加一个值。

    const items = [];
    // bad
    items[items.length] = 'string';
    // good
    items.push('string');

  • 使用扩展运算符做数组浅拷贝，类似上面的对象浅拷贝。

    const items = [];
    // bad
    const len = items.length;
    const itemsCopy = [];
    for (let i = 0; i < len; i++) {
    itemsCopy[i] = items[i];
    }
    // good
    const itemsCopy = [...items];

  • 获取数组的元素时，在合适的情况下，优先使用数组解构赋值。

    const arr = [1, 2, 3, 4];
    // bad
    const first = arr[0];
    const second = arr[1];
    // good
    const [first, second] = arr;

  • 将一个可迭代的对象转换成数组，用 ... 运算符比 Array.from 更好。

    const foo = document.querySelectorAll('.foo');
    // good
    const nodes = Array.from(foo);
    // best
    const nodes = [...foo];

  • 将一个类数组对象转成一个数组，用 Array.from 。

    const arrLike = { 0: 'foo', 1: 'bar', 2: 'baz', length: 3 };
    // bad
    const arr = Array.prototype.slice.call(arrLike);
    // good
    const arr = Array.from(arrLike);

字符串#

  • 字符串应使用单引号 ''。

    // bad
    const name = "Jack Sparrow";
    // bad - 模板字符串应该包含插入文字或换行
    const name = `Jack Sparrow`;
    // good
    const name = 'Jack Sparrow';

  • 当需要动态生成字符串时，使用模板字符串而不是字符串拼接。

    // bad
    function sayHi(name) {
    return 'How are your, ' + name + '?';
    }
    // bad
    function sayHi(name) {
    return ['How are your, ', name, '?'].join();
    }
    // good
    function sayHi(name) {
    return `How are you, ${name}`;
    }

  • 永远不要使用 eval() 。

  • 不要使用不必要的转义符。

    // bad
    const name = '\"Jack Sparrow\"';
    // good
    const name = '"Jack Sparrow"';
    // best
    const saySomething = `my name is '${name}'`;

函数#

  • 使用命名函数表达式而不是函数声明。

    函数声明会发生提升；匿名函数表达式会导致难以追踪错误。

    // bad - 函数声明会发生提升
    function example() {
    // ...
    }
    // bad - 匿名函数表达式会导致难以追踪错误
    const example = function() {
    // ...
    }
    // good
    const example = function showExample() {
    // ...
    }

  • 不要使用 arguments 命名参数。

    它的优先级高于每个函数作用域自带的 arguments 对象，这会导致函数自带的 arguments 值被覆盖。

    // bad
    const example = function showExample(name, options, arguments) {
    // ...
    }
    // good
    const example = function showExample(name, options, args) {
    // ...
    }

  • 用默认参数语法，而不是在函数里对参数重新赋值。

    // bad
    const example = function showExample(opts) {
    opts = opts || {};
    // ...
    }
    // good
    const example = function showExample(opts = {}) {
    // ...
    }

  • 把默认参数放在最后。

    // bad
    const example = function showExample(opts = {}, name) {
    // ...
    }
    // good
    const example = function showExample(name, opts = {}) {
    // ...
    }

  • 不要修改参数，不要对参数重新赋值。

    操作参数对象对原始调用者会产生意想不到的副作用。不要改变参数的数据结构，保留参数原始值和数据结构。

    // bad
    const example = function showExample(opts = {}) {
    opts.key = 1;
    // ...
    }
    // good
    const example = function showExample(opts = {}) {
    const key = opts.key ? opts.key : 1;
    // ...
    }
    // bad
    function example(a) {
    a = 1;
    // ...
    }
    // good
    function example(a) {
    const b = a || 1;
    // ...
    }

  • 函数定义部分要有空格。

    // bad
    const a = function(){}
    const a = function() {}
    const a = function (){}
    // good
    const a = function () {}
    const example = function showExample() {}

  • 函数包含多个返回值时，优先使用对象，而不是数组。

    // bad
    function example() {
    // ...
    return [left, right, top, bottom];
    }
    // 使用时，需要严格对照返回值的顺序
    const [left, _, top] = example();
    // good
    function example() {
    // ...
    return {left, right, top, bottom};
    }
    const {left, top} = example();

箭头函数#

  • 在回调函数里，使用箭头函数。

    // bad
    [1, 2, 3].map(function (x) {
    const y = x + 1;
    return x * y;
    });
    // good
    [1, 2, 3].map((x) => {
    const y = x + 1;
    return x * y;
    });
    // best
    [1, 2, 3].map((x) => x * (x + 1));

  • 箭头函数的参数总是使用小括号包裹在。

    这样做使代码更清晰一致，并且当想要添加或删除参数时比较省事。

    // bad
    [1, 2, 3].map(x => x * x);
    // good
    [1, 2, 3].map((x) => x * x);

  • 使箭头函数体有一个清晰的返回。

    // bad
    [1, 2, 3].map((x) =>
    x * x);
    [1, 2, 3].map((x) =>
    (x * x));
    // good
    [1, 2, 3].map((x) => x * x);
    [1, 2, 3].map((x) => (
    x * x
    ));

比较运算符#

  • 使用严格运算符，用 === 和 !== ，而不是 == 和 != 。

    // bad
    if (id == 1) {
    // ...
    }
    // good
    if (id === 1) {
    // ...
    }

  • 布尔值要用缩写，而字符串和数字要明确使用比较运算符。

    // bad
    if (isValid === true) {
    // ...
    }
    if (name) {
    // ...
    }
    if (items.length) {
    // ...
    }
    // good
    if (isValid) {}
    if (name !== '') {}
    if (items.length > 0) {}

  • 三元表达式不应嵌套，通常是单行表达式，如果涉及多个判断，请改用控制语句 if 或 switch 。

    // bad
    const name = a > b ? 'string1' : x > y ? 'string2' : 'string3';
    // good
    const name = a > b ? 'string1' : 'string2';
    // good
    let name;
    if (a > b) {
    name = 'string1';
    } else if (x > y) {
    name = 'string2';
    } else {
    name = 'string3';
    }

  • 避免使用不必要的三元表达式。

    // bad
    const str = a ? a : b;
    const isValid = a ? true : false;
    const isValid = a ? false : true;
    // good
    const str = a || b;
    const isValid = !!a;
    const isValid = !a;

  • 用圆括号组合操作符。只有当标准的算术运算符（ + 、 - 、 * 或 / ）且优先级明显时，才可以不用圆括号括起来。

    // bad
    if (a || b && c) {
    // ...
    }
    // good
    if (a || (b && c)) {
    // ...
    }
    // good
    const res = a + b * c;

注释#

  • 单行注释用 // ，将单行注释放在被注释区域上面。

  • 多行注释用 /** ... */ 。

  • 所有注释开头都空一格，方便阅读。

    // bad
    //这里是单行注释
    const isValid = true;
    /**
    *这里是多行注释
    */
    const example = function showExample() {}
    // good
    // 这里是单行注释
    const isValid = true;
    /**
    * 这里是多行注释
    */
    const example = function showExample() {}

空格#

  • 一个缩进使用两个空格。

    // bad
    const example = function showExample() {
    console.log('...');
    }
    // good
    const example = function showExample() {
    console.log('...');
    }

  • 在大括号前面空一格。

    // bad
    const example = function showExample(){}
    // good
    const example = function showExample() {}

  • 在控制语句（ if 、 while 等）的圆括号前空一格。在函数调用和定义时，函数名与参数列表之间不空格。

    // bad
    if(isValid) {
    example ();
    // ...
    }
    // good
    if (isValid) {
    example();
    // ...
    }
    // bad
    function example () {}
    // good
    function example() {}

  • 运算符之间使用一个空格隔开。

    // bad
    const res=x+y;
    // good
    const res = x + y;

  • [ 后和 ] 前不使用空格。逗号分隔符（ , ）后使用一个空格。

    // bad
    const arr = [ 1, 2, 3 ];
    console.log(arr[ 0 ]);
    // good
    const arr = [1, 2, 3];
    console.log(arr[0]);

  • { 后和 } 前使用一个空格。分隔符（ , 和 : ）后使用一个空格。

    // bad
    const obj = {a:1,b:2};
    // good
    const obj = { a: 1, b: 2 };
    // bad
    function example() {console.log('...');}
    // good
    function example() { console.log('...'); }

  • 合理使用空行。

命名规范#

  • 避免使用一个字母或无意义的名称命名，尽量让命名有意义。

    // bad
    const arr = [];
    function q() {}
    const temp = true;
    // good
    const items = [];
    function query() {}
    const hasVal = true;

  • 使用小驼峰命名法来命名对象、函数、实例。

    // bad
    const my_object = {};
    function MyFunction() {}
    // good
    const myObject = {};
    function myFunction() {}

  • 使用大驼峰命名法来命名类。

    // bad
    class point {}
    // good
    class Point {}

  • 不要使用前置或后置下划线。

    // bad
    const _firstName = '';
    const __firstName__ = '';
    // good
    const firstName = '';

  • 保持文件名与导出模块（ export default ）的名字相同。导入（ import ）的时候参数也叫这个名字。

  • 当导出（ export default ）一个函数时，函数名使用小驼峰。当导出（ export default ）一个类或组件（ React.Component ）时，使用大驼峰命名。
