风格指南

这里是 ConciseJS 框架特有的风格指南。它是在现有语言规范（ JavaScript 、 React & JSX 、 CSS & Sass ）基础之上，结合框架制订而成。可以最大化的帮助开发者规避某些错误、确保项目的一致性和代码的可读性。

在开始了解框架风格指南之前，开发者需先了解以下具体规范：

  • JavaScript 风格指南
  • React/JSX 风格指南
  • CSS/Sass 风格指南

命名#

组件#

  • 用 .jsx 作为组件扩展名。
  • 用大驼峰作为文件名，且文件名与导出模块（ export default ）的名字相同。

props#

  • 避免与 HTML 的属性名相同。

    // bad
    <MyComponent style="dark" />
    // good
    <MyComponent mode="dark" />

  • prop 的定义应该尽量详细，至少需要指定其类型。

  • 使用小驼峰命名法命名 prop 。
