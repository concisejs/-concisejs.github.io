约定式路由

ConciseJS 项目支持约定式路由，约定式路由也叫基于文件系统的路由，即不需要手写配置，通过分析项目目录和文件自动生成路由配置。

约定式路由的权限低于配置式路由，当没有配置式路由文件时， ConciseJS 会启用约定式路由模式。

路由规则#

文件被注册为路由，需满足以下所有规则：

  • src/pages 目录下。
  • 以 .js 、.jsx 为后缀的文件，建议都以 .jsx 后缀命名文件。
  • 以大驼峰命名的文件，或以大驼峰命名的文件夹下的 index 文件。

满足以下任意规则，文件会被忽略注册：

  • src/pages/** 目录下 components 文件夹下的文件。
  • _ 开头命名的文件。

动态路由#

约定 [] 包裹的文件或文件夹为动态路由，比如：

  • src/pages/trade/detail/[id].jsx 会被注册为路由 /trade/detail/:id
  • src/pages/trade/[id]/detail.jsx 会被注册为路由 /trade/:id/detail

示例#

假设有以下目录及文件：

Face/src/pages/
|-- Home.jsx
|-- Battle.jsx
|-- Equipment.jsx
|-- compariosn/
| |-- IdentityQuery.jsx
| |-- TrackQuery.jsx
| |-- One2one.jsx
| |-- N2n.jsx
| |-- compare-list/
| | |-- [id].jsx
|
|-- library/
| |-- Face.jsx
| |-- DynamicCapture.jsx
| |-- upload-task-list/
| | |-- [id].jsx
|
|-- deploy/
| |-- Add.jsx
| |-- List.jsx
| |-- Alarm.jsx
| |-- History.jsx
| |-- AlarmDetail.jsx

则生成的路由为：

const routes = [
// 首页
{path: '/home'},
// 战果展示
{path: '/battle'},
// 设备运维
{path: '/equipment'},
// 比对查询
{
name: 'compariosn',
routes: [
// 比对查询 - 身份查询
{path: '/comparison/identity-query'},
// 比对查询 - 轨迹查询
{path: '/comparison/track-query'},
// 比对查询 - 1:1比对
{path: '/comparison/one2one'},
// 比对查询 - n:n比对
{path: '/comparison/n2n'},
// 比对查询 - 比对列表
{path: '/comparison/compare-list/:id'}
]
},
// 人脸库
{
name: 'library',
routes: [
// 人脸库 - 静态人脸库
{path: '/library/face'},
// 人脸库 - 动态抓拍库
{path: '/library/dynamic-capture'},
// 人脸库 - 上传任务列表
{path: '/library/upload-task-list/:id'}
]
},
// 布控稽查
{
name: 'deploy',
routes: [
// 布控稽查 - 新建布控
{path: '/deploy/add'},
// 布控稽查 - 布控明细
{path: '/deploy/list'},
// 布控稽查 - 布控告警台
{path: '/deploy/alarm'},
// 布控稽查 - 告警历史
{path: '/deploy/history'},
// 布控稽查 - 告警详情
{path: '/deploy/alarm-detail'}
]
}
]
